Snap! プログラミング

　Snap! は Scratch の皮を被った Scheme と言われています。Scheme は Lisp 系のプログラミング言語ですが、意識する必要はありません。Scratch を知っているならば馴染みのあるブロックで同じようにプログラミングできます。Scratch のデフォルトのスプライトはネコですが、Snap! は矢印になっています。中学生以上の人には受け入れやすいでしょう。Scratch にはない、なんじゃこりゃというブロックもあると思いますが、あまり気にせずに興味をもったところから理解していければいいのではないでしょうか。Scratch ではできなかったようなアルゴリズムを表現することができます。

  • Snap!(Build Your Own Blocks) が表すように、自作ブロックが作りやすくなっています。Scratch のブロック定義は、ある動作をするスクリプトを一つのブロックで表す、サブルーチンや手続きという役割でした。 引数として入力できるのは数値、テキスト、真偽値だけです。Snap! では、「動き」「見た目」「音」... 8個すべてのカテゴリーのブロックを定義することができます。独自のカテゴリーの作成も可能です。演算の値を返すような楕円形のブロックや if ブロックの条件判断などに使用するひし形のブロック、自分好みの if 型ブロックや繰り返し処理を行うブロックまでも作れます。引数にブロックやスクリプトを入力することもできます。関数の引数や戻り値に関数を指定できるものを高階関数と言い、それにあたります。無名関数（ラムダ関数）のような機能も使うことができます。ラムダはギリシャ文字の λ で表されて Snap! のロゴに表示されています。
  • Scratch では、定義できる変数はグローバルなものかそのスプライトだけで使用するものの二種類でした。Snap! にはスクリプト変数というちゃんとしたローカル変数があり、変数を生成するブロックをスクリプト内に置くことができます。ブロック定義内でも作成できるのでやりやすくなっています。Snap! には〔for〕〔for each〕ループがあり、ループでのカウンターやリスト要素の参照に使用できる備え付けの変数があるので、自分でそのための変数を作成する必要はありません。
  • Snap! では APL や R や Julia 言語などのように、リスト全体に対して演算をしたり（[リスト] * 10）、リスト同士で演算をしたり（[リスト1] * [リスト2]）することができます。Lisp は LISt Processing が由来であり、Snap! もリスト操作のためのブロックが充実しています。
  • Snap! はデバッグ機能が充実しています。〔pause all〕ブロックをチェックしたい箇所に入れれば一時停止させることができます。再開ボタンがあるので続行可能です。デバッグモードにするボタンがあり、実行速度を調整することができます。その際は、実行中のブロックをハイライトしたり、ブロック中の変数やリストの値を表示してくれます。
  • Scratch では他のプログラムで作成したスクリプトを利用する時は「バックパック」を使ってやり取りしていました。Snap! では「エクスポート」「インポート」で書き出し読み込みができます。こちらはクラウドではなくパソコンに保存されます。
  • Scratch では、ターボモードにすると高速で実行させることができました。しかしそれはプログラム全体に対して作用するものでした。Snap! ではスクリプト中でターボモードをオンオフできますし、〔warp〕という指定されたスクリプトを高速実行するブロックもあります。
  • Scratch では、背景に文字を描画するにはコスチュームエディターで作成するしかありません。Snap! ではブロックを使ってプログラム実行中に指定の文字列を描画することができます。また、スクリプトで描画した文字列を含むものをコスチュームとして利用することができます。手で描くことが苦手な人でもきちんとしたコスチュームを作ることができます。
  • Scratch では、値を返す定義ブロックが作れませんでした。そのため再帰呼出しで階乗やフィボナッチ数列を求めるプログラムは作れませんでした。Snap! ではそれができます。Scheme などでは繰り返し処理をおこなうのにも再帰呼出しを使用します。Snap! でも再帰呼出しで繰り返し処理をすると高速なプログラムを作ることができます。
  • Snap! ではブロック中に JavaScript コードを書くことができます。変数やリストで値を送り、JavaScript で処理をした値を受け取ることができます。
このように説明すると Scratch を凌駕しているように思えますが、実行スピードは劣りますし Scratch のスクリプトを移植しても同じようには動作しない場合もあります。ゲームのようなものを作るには Scratch のほうに分があるように思えます。ただ、プログラミングという点ではステップ実行などのデバッグ環境や高度なブロックが使え、関数型プログラミングも体験できる Snap! におもしろみを感じられます。

ビジュアルプログラミング

　プログラムは一般的には英語のような命令を連ねて作成されます。個々の命令や構文規則について把握している必要がありますし、プログラムは一字一句の入力ミスでエラーになります。ブロックなどにした命令をつなげてプログラムを作成し、変数の値などや実行結果も画面上に表示されるようにしたものがビジュアルプログラミングと言われる Scratch や Snap! です。使用できるブロックはすべてブロック選択エリアにあるので、そこから目的に合ったものを選んで作っていきます。プログラム実行の主役がコスチュームで表されるスプライトです。 コンピューターらしい計算などをさせることもできますが、スプライトを画面上で動かしてみることで自分が作ったプログラムが正しく動作するか確認できるので入門者には分かりやすいと思います。 とっつきやすい反面、プログラムをブロックとして画面上に表示される範囲でしか見られないので全体が把握しづらかったり、変数やブロックをどこで使用しているかを検索できないことなど使いづらい点もあります。ですが、プログラミングの練習や実験、プロトタイピングツールとしても十分有益です。

　Scratch については書籍も豊富に出回っていて Web での情報も多いのですが、Snap! については日本語での情報が少ないのが現状です。公式のマニュアルを読むのが一番だと思いますが、副読本として理解の一助になれば幸いです。Web ブラウザが対応していれば、PDF ファイルへのリンクを左クリックすることで見ることができます。使い方を知りたいところや興味のあるところだけを参照するというスタンスでどうぞ。加筆修正中ですので最新版をご利用ください。

「作図チャレンジ」を使ったプログラミング入門

　ステージ上に簡単な図形を描くことで、プログラミングの基本である「順次実行」「条件分岐」「繰り返し」を体験します。 Snap! 入門と言うよりはプログラミング入門を意図しているので、むしろ Snap! にもともと備わっているものではないブロックを使ってプログラミングします。ステージに描かれた図形に沿ってスプライトが動くように、自分で考えてブロックを組み立ててもらうパズル集のようなものです。「階段」を描くのは簡単ですが奥が深い問題で、これを突き詰めていくと「迷路」の問題につながります。
